# PRD — Mac CCTV: 자리 비움 도난 감시 앱

- 작성일: 2026-07-04
- 상태: Draft v2 (미확정 사항 §13 참고)

---

## 1. 개요

**문제.** 카페·도서관·공항에서 노트북을 켜둔 채 잠깐 자리를 비울 때(화장실, 주문 등), "누가 내 노트북을 가져가면 어떡하지"라는 불안 때문에 마음 편히 자리를 뜨지 못한다. 짐을 다 싸서 들고 가거나, 옆 사람에게 부탁하는 것이 현재의 대안이다.

**이 제품이 실제로 파는 것.** 실제 도난은 드물지만, 불안은 자리를 비울 때마다 발생한다. 따라서 이 제품의 본질은 도난 방지 장치가 아니라 **"마음 편히 15분 자리를 비울 수 있는 자유"를 파는 보험형 안심 상품**이다. 카피·성공 지표·수익화 판단은 모두 이 정의를 기준으로 한다 — 핵심 가치 지표는 도난 검거가 아니라 "안심하고 자리를 비운 횟수"다.

**솔루션 한 줄.** Mac에서 단축키 한 번으로 웹캠 감시를 켜고, iPhone에서 실시간(1~2초 지연)으로 내 자리를 지켜보며, 모든 영상은 내 iCloud에 자동 저장되어 7일 뒤 삭제된다. 화면에는 아무것도 표시되지 않는다.

**핵심 가치 (우선순위 순).**
1. **안심 (일상 가치)** — 자리를 비우는 동안 폰으로 내 자리가 보인다. 매 세션 발생하는 1순위 가치.
2. **조기 인지 + 원격 대응 (사건 시 가치)** — 이상 징후를 알림으로 받고, 사이렌으로 도난을 현장에서 중단시킨다.
3. **증거 (최후의 가치)** — 도난이 완료되어도 사건 순간까지의 영상이 이미 클라우드에 있다.
4. **마찰 제로** — 단축키 한 번(켜기), 한 번(끄기). 회원가입 없음, 설정 없음, 운영비 없음.

## 2. 타겟 사용자 & 시장

- **페르소나 (상황 기반) 3개:**
  1. **카공족 대학생** — 도서관·카페, 주 5회 이상 자리 비움. 최다 빈도 세그먼트이자 바이럴의 진원지.
  2. **노트북이 유일한 생산수단인 프리랜서·디지털 노마드** — 해외 카페, 도난 = 생계 중단. 절실함 최고, 입소문 전도사 세그먼트.
  3. **출장 직장인** — 공항 라운지·호텔 로비. 구매력 최고, 향후 수익화 1순위 대상.
- **여성 안전 앵글**: 별도 세그먼트가 아니라 위 페르소나들의 마케팅 톤·채널 선정(여성 커뮤니티, 안전 크리에이터 협업)에 반영. 혼자 작업하는 상황에서 자리 비움 불안이 더 크고 안전 앱 수용도가 높다.
- **지역**: 글로벌 우선 (영어 기본 + 한국어). 한국은 치안이 좋아 니즈가 상대적으로 약하고, 북미·유럽·동남아 카페 문화권에서 노트북 도난이 실제로 빈번하다.
- **기기**: MacBook 사용자 = 고가 기기 소유자 = 도난 불안이 크고 iPhone 보유율이 높다 (Mac+iPhone 조합 전제가 유효).
- **바이럴 포인트**: "화장실 갈 때 이거 켜고 가세요" — 컨셉이 한 문장으로 설명되고 데모 영상이 재미있어 숏폼(TikTok/Reels/Shorts)과 잘 맞는다.

## 3. 경쟁 서비스 분석

| 서비스 | 방식 | 실시간 영상 | 클라우드 저장 | 가격 | 비고 |
|---|---|---|---|---|---|
| **MeerCOP** (한국) | 노트북↔폰 연결, 움직임 감지 경보, 원격 카메라, GPS | 부분적 (원격 촬영 위주) | 사진 증거 중심 | 무료 체험 후 유료 | 가장 유사한 포지션. 상시 실시간 스트리밍보다 경보 중심 |
| **Do Not Disturb** (Objective-See) | 뚜껑 열림 감지 → iOS 앱 알림, 원격 사진 촬영 | ✕ (사진) | ✕ | Mac 무료 / iOS 구독 | "evil maid" 방어 컨셉. 현재 유지보수 중단 상태 |
| **Unplug Alarm** (Mac App Store) | 뚜껑 닫힘/전원 분리 시 알람 + 웹캠 사진 | ✕ | ✕ | 유료 앱 | 알람(소리) 중심, 폰 연동 없음 |
| **iAlertU** | 움직임/키 입력 시 알람 + 사진 이메일 전송 | ✕ | ✕ (이메일) | 무료 | 오래된 앱, Apple Remote 시대 유물 |
| **Prey** | 기기 위치 추적 + 원격 잠금/촬영 (도난 **후** 회수) | ✕ | 자사 서버 | 무료 3대 / 구독 | B2B 기기 관리 성격. 사전 감시가 아닌 사후 추적 |
| **Alfred / 와치두잉 / Presence** | 여분 기기를 CCTV로 활용 | ○ | 구독 시 | freemium | 카테고리가 다름: 공간 감시용 CCTV. "내 노트북 지키기" 시나리오 아님 |

**시장의 빈 자리 (우리의 차별점):**
1. 기존 앱은 전부 **"사진 + 경보"** 다. **"내 자리를 실시간 영상으로 지켜본다"**는 경험을 주는 제품이 없다.
2. 기존 앱은 증거가 기기 안이나 이메일에 남는다. **도난 순간까지의 영상이 클라우드에 실시간 백업**되는 제품이 없다 — 도둑이 기기를 가져가면 증거도 함께 사라지는 구조적 결함.
3. 기존 앱은 설정이 많고 UI가 낡았다. **단축키 하나로 켜고 끄는 제로-마찰 UX**가 없다.
4. Prey류는 도난 **후** 회수 도구다. 우리는 도난 **전·중** 감시와 억제에 집중한다 (상호보완적 — 경쟁 아님).

참고: [MeerCOP](https://meercop.com/) · [Do Not Disturb](https://digitasecurity.com/donotdisturb/) · [Unplug Alarm](https://apps.apple.com/us/app/unplug-alarm-anti-theft-app/id1489026302) · [iAlertU](https://www.macupdate.com/app/mac/21282/ialertu) · [Prey](https://preyproject.com/)

## 4. 핵심 사용자 시나리오

**시나리오 A — 카페에서 화장실 (메인 플로우)**
1. 카페에서 작업 중. 화장실에 가고 싶다.
2. `⌃⌘C` (기본 단축키) 를 누른다. 메뉴바 아이콘이 조용히 점등 상태로 바뀐다. 화면에는 아무 창도 뜨지 않는다.
3. 화면을 잠그고(습관) 자리를 뜬다.
4. 화장실에서 iPhone 앱을 연다 → 1~2초 지연의 실시간 영상으로 내 자리가 보인다.
5. 돌아와서 같은 단축키를 누른다 → 감시 종료. 녹화본은 iCloud에 7일간 보관.

**시나리오 B — 실제 도난 시도**
1. 감시 중, 누군가 노트북 앞에 나타나 화면 앞에서 움직인다 → iPhone에 푸시 알림 "움직임 감지됨".
2. 도둑이 뚜껑을 닫는다 → 닫히기 직전까지의 영상 조각이 즉시 iCloud로 플러시되고, "뚜껑 닫힘" 긴급 알림 발송.
3. 도둑이 기기를 들고 이동 → **자동 사이렌 발동**: Mac이 최대 음량 경보 + 전체 화면 "이 기기는 녹화·추적 중" 경고. 사용자도 iPhone에서 수동으로 사이렌을 켤 수 있다. 카페 전체의 시선이 집중되어 절도 중단 유도.
4. 그래도 노트북을 가져가면, **얼굴이 찍힌 영상은 이미 사용자 iCloud에 있다.** iPhone 앱에서 바로 재생·저장·신고 가능.

**시나리오 C — 아무 일도 없음 (99.9%의 경우)**
1. 켜고, 다녀오고, 끈다. 앱은 아무것도 묻지 않고 아무것도 보여주지 않는다.
2. 7일 뒤 영상은 자동 삭제된다. 사용자가 관리할 것이 없다.

## 5. 요구사항 (MoSCoW)

### Must (MVP)
- **M1. 전역 단축키 토글**: 어떤 앱을 쓰고 있어도 단축키로 감시 시작/종료. 기본값 **⌃⌘C** (Control+Command+C), 커스터마이징 가능.
- **M2. 무화면 녹화**: 감시 중 어떤 프리뷰 창·오버레이도 표시하지 않는다. 메뉴바 아이콘 상태 변화만 존재. (참고: 웹캠 녹색 LED는 하드웨어 제약상 켜진다 — §8.1)
- **M3. 실시간 스트리밍**: iPhone에서 1~2초 지연으로 시청 (WebRTC P2P). P2P 실패 시 클라우드 청크 스트리밍(10~20초 지연)으로 자동 폴백.
- **M4. 클라우드 녹화**: 감시 시작부터 종료까지 전 구간을 4~6초 청크로 사용자 iCloud(CloudKit private DB)에 업로드. 네트워크 단절 시 로컬 버퍼링 후 재업로드.
- **M5. 7일 자동 삭제**: 7일 경과 청크는 Mac/iPhone 앱 실행 시 자동 정리.
- **M6. 이벤트 감지 & 알림** — 4계층 감지 스택, 감지 시 iPhone 푸시 알림:
  - ① **기기 터치**: 감시 중 키보드·트랙패드·마우스 입력 발생 (시스템 유휴시간 리셋 감지 — 공개 API, 권한 불필요)
  - ② **전원 분리**: 충전 케이블 뽑힘
  - ③ **뚜껑 닫힘**
  - ④ **카메라 모션**: 프레임 *일부* 움직임 = "사람 접근", 프레임 *전체* 시프트 = "기기가 움직여짐"으로 구분 분류
  - 참고: 최신 MacBook에는 공개 API로 접근 가능한 가속도계가 없다 (구형 Sudden Motion Sensor는 SSD 전환과 함께 제거됨). "기기 이동" 감지는 ①+④ 조합으로 근사하며, Apple Silicon의 비공개 가속도계 활용은 실험 기능(C5)으로 분리.
- **M7. 잠자기 방지**: 감시 중 시스템 잠자기 차단 (화면 잠금·디스플레이 꺼짐은 허용).
- **M8. 긴급 플러시**: 뚜껑 닫힘·전원 분리·강제 종료 감지 시 진행 중 청크를 즉시 업로드.
- **M9. 사이렌** — 두 가지 발동 경로:
  - **수동**: iPhone 라이브 화면의 사이렌 버튼(길게 누르기) → Mac 최대 음량 경보 + 전체 화면 "이 기기는 녹화·추적 중" 경고.
  - **자동 (보수적 트리거)**: "기기가 들려 이동 중"이 확실할 때만 발동 — 전역 프레임 시프트가 수 초 이상 지속되고 복합 신호(입력 활동·전원 분리 등)가 동반될 때. 단순 접근이나 테이블 흔들림으로는 절대 발동하지 않는다. 발동 즉시 iPhone 긴급 알림 병행.
  - **해제**: Mac에서 감시 종료 단축키(+인증)로 3초 안에 해제 가능해야 한다.

### Should (MVP 직후)
- S1. 세션 타임라인: iPhone에서 지난 7일 세션 목록·재생·기기 저장.
- S2. 움직임 감지 구간 하이라이트 표시.
- S3. 감시 종료 시 비밀번호/Touch ID 요구 옵션 (도둑이 앱을 끄는 것 방지).

### Could (v2+)
- C1. 웹 뷰어 (링크로 친구에게 잠깐 보여주기).
- C2. 여러 Mac 지원.
- C3. Apple Watch 컴플리케이션 (손목에서 바로 확인).
- C4. 소리 감지, 마이크 녹음 옵션 (기본 꺼짐 — §10 법적 이슈).
- C5. (실험적) Apple Silicon 비공개 MEMS 가속도계 기반 정밀 이동 감지 — IOKit HID로 접근 가능함이 확인되어 있으나 비공개 API라 App Store 거절 위험. 모듈로 격리해 직접 배포 빌드에서만 활성화하는 방안 검토.

### Won't (명시적 제외)
- 위치 추적/GPS (Find My가 이미 함), 원격 잠금/삭제 (Prey 영역), Windows 지원, 개발자 서버에 영상 저장 (절대 안 함 — 프라이버시 + 비용).

## 6. UX 설계 — "설정 없는 앱"

### Mac 앱 (메뉴바 상주, Dock 아이콘 없음)
- **상태 2개뿐**: 대기(외곽선 아이콘) / 감시 중(채워진 아이콘). 감시 중에도 눈에 띄는 변화 최소화.
- **메뉴바 클릭 시 작은 팝오버 1장**:
  - 상태 텍스트 ("감시 중 · 12분 · iPhone 연결됨")
  - 시작/종료 버튼 (단축키 안내 병기)
  - 설정 3개만: 단축키 변경 / 화질 (저·중·고) / 알림 on-off
- **온보딩 = 1회, 3단계**: ① 카메라 권한 → ② iCloud 확인 → ③ "iPhone에 컴패니언 앱을 설치하세요" QR. 끝.

### iPhone 앱 (화면 2개뿐)
- **화면 1 — 라이브**: 앱을 열면 바로 실시간 영상. 상단에 연결 상태·경과 시간, 하단에 **사이렌 버튼**(실수 방지를 위해 길게 누르기). 그 외 UI 없음.
- **화면 2 — 보관함**: 세션 목록 (날짜·길이·이벤트 뱃지) → 탭하면 재생. 좌측 스와이프로 삭제. "7일 뒤 자동 삭제" 안내 문구.
- 로그인 화면 없음 (같은 Apple ID면 자동 연결).

### 톤 & 브랜딩
- 이름: **Mac CCTV** (브랜드 컨셉). App Store 표기명은 **"CCTV for Mac"** 확정 — Apple 상표 정책상 "Mac" 접두어는 심사 거절 사례가 있고, "for Mac" 형태는 가이드라인이 명시적으로 허용. 도메인·마케팅에서는 Mac CCTV 컨셉 그대로 사용.
- 무드: 보안 앱의 위압감 대신 가볍고 위트 있게. "커피 식기 전에 다녀오세요 ☕️" 같은 카피.

## 7. 아키텍처

```
┌─────────── Mac 앱 (Swift/SwiftUI, 메뉴바) ───────────┐
│ AVFoundation 캡처 (720p, ~800kbps H.264, 오디오 없음)  │
│   ├─→ ① WebRTC 트랙 ──(P2P)──→ iPhone 라이브 시청     │
│   └─→ ② AVAssetWriter 4~6초 fMP4 청크                │
│          └─→ 로컬 버퍼 → CloudKit private DB 업로드    │
│ 이벤트 감지: 입력터치·전원·뚜껑·카메라모션 → 푸시       │
└──────────────────────────────────────────────────────┘
                          │
              사용자 본인의 iCloud (CloudKit)
              - Session / Chunk / Event / Signal 레코드
              - 시그널링: SDP·ICE를 CKRecord로 교환,
                CKSubscription 사일런트 푸시로 상대에게 전달
                          │
┌─────────── iPhone 앱 (SwiftUI) ──────────────────────┐
│ 라이브: WebRTC 수신 (실패 시 청크 폴백 재생)            │
│ 보관함: 청크를 AVQueuePlayer로 이어 재생               │
│ 알림: CKSubscription 푸시 (APNs — CloudKit 경유 무료)  │
└──────────────────────────────────────────────────────┘
```

**핵심 설계 결정과 근거:**

1. **시그널링 서버 없음** — WebRTC 연결 수립에 필요한 SDP/ICE 교환을 CloudKit 레코드 + 사일런트 푸시로 처리. 개발자 서버 0대.
2. **STUN은 무료 공용 서버** (Google/Cloudflare STUN). **TURN 릴레이는 두지 않는다** (비용 발생 지점) — 대칭 NAT 등으로 P2P가 뚫리지 않으면 M3의 폴백(클라우드 청크 스트리밍, 10~20초 지연)으로 자동 전환. 사용자에게는 "지연 모드"로 표시.
3. **저장은 전적으로 사용자 iCloud** — CloudKit private DB 용량은 사용자 본인의 iCloud 쿼터에서 차감. 개발자에게 저장 비용이 전가되지 않고, 프라이버시 논쟁도 원천 차단 ("영상은 당신의 iCloud에만 있습니다").
4. **7일 삭제는 클라이언트 주도** — CloudKit에 TTL이 없으므로 Mac/iPhone 앱이 실행될 때마다 만료 청크를 삭제. 두 앱 모두 장기간 실행되지 않으면 삭제가 지연될 수 있으나, 본인 저장공간이므로 허용 가능한 트레이드오프.
5. **화질 기본값 720p/800kbps** ≈ 시간당 약 350~400MB. 무료 iCloud 5GB 사용자도 하루 30분씩 7일 보관 시 ~1.4GB로 수용 가능. 저화질 옵션(480p)으로 절반 이하 절감.

## 8. 제약 및 엣지 케이스

| # | 상황 | 대응 |
|---|---|---|
| 8.1 | **웹캠 녹색 LED** | 하드웨어 제약으로 끌 수 없음. **끄려고 시도하지 않는다** (보안 정책 위반 + 심사 거절 사유). 오히려 "녹화 중임이 보이는 것이 도난 억제 효과"로 포지셔닝. 화면에 아무것도 안 뜨는 것(M2)은 문제없이 가능 |
| 8.2 | 도둑이 뚜껑을 닫음 | 닫힘 이벤트 → 긴급 플러시(M8) + 알림. 닫힌 후에는 카메라가 가려지므로 녹화 중단. **닫히기 전 영상이 증거** |
| 8.3 | 도둑이 앱을 강제 종료 | 종료 시점까지의 청크는 이미 iCloud에 있음. S3(종료 시 인증)로 추가 방어. 메뉴바 앱이라 일반인은 끄는 법을 잘 모름 |
| 8.4 | 카페 Wi-Fi 불안정/캡티브 포털 | 로컬 링버퍼에 청크 보관 → 네트워크 복구 시 순서대로 업로드. 라이브는 폴백 모드 안내 |
| 8.5 | iCloud 용량 부족 | 감시 시작 시 여유 용량 확인, 부족하면 화질 자동 하향 + 경고. 업로드 실패 시 로컬 보관 후 안내 |
| 8.6 | 화면 잠금 상태 | 백그라운드 캡처는 화면 잠금과 무관하게 동작. 사용자에게 "잠그고 다녀오세요" 권장 (온보딩 카피) |
| 8.7 | 배터리 | 감시 중 시스템 잠자기만 차단, 디스플레이는 끔 → 30분 감시 시 배터리 영향 수 % 수준 목표 |
| 8.8 | iPhone이 셀룰러망 | P2P가 CGNAT에 막힐 확률 높음 → 폴백 모드가 주 경로가 될 수 있음을 UX에 반영 ("지연 약 15초") |
| 8.9 | "기기 이동" 감지 정밀도 | 공개 가속도계 부재 → 입력 터치(①) + 전역 프레임 시프트(④) 조합으로 근사. 테이블 흔들림 등 오탐은 민감도 임계값으로 조정 |
| 8.10 | 자동 사이렌 오탐 | 보수적 복합 트리거(M9)로 최소화. 발동 시 Mac에서 단축키+인증으로 3초 내 해제. **공공장소 오탐 1회 = 앱 삭제**라는 전제로 임계값을 정하고, 확신이 낮으면 사이렌 대신 iPhone 알림만 보낸다 |

## 10. 법적·심사 리스크

1. **App Store 심사**: "몰래 녹화" 프레이밍은 거절 사유가 될 수 있음 → **"내 기기·내 자리의 보안 감시"**로 명확히 포지셔닝. 카메라 사용 목적 문구(NSCameraUsageDescription)와 마케팅 모두 anti-theft로 일관. LED를 우회하지 않으므로 기술적 문제 없음.
2. **음성 녹음**: 한국(통신비밀보호법)·미국 일부 주(two-party consent) 등에서 제3자 대화 녹음은 불법 소지 → **오디오는 기본 꺼짐, MVP에서는 아예 제외** (C4).
3. **공공장소 촬영**: 본인 기기에서 본인 자리를 촬영하는 것은 일반적으로 CCTV·가정용 캠과 동일한 지위. 다만 앱 내에 "각 지역 법규를 확인하세요" 고지 포함.
4. **프라이버시 라벨**: 영상이 개발자 서버를 거치지 않음 → App Store 프라이버시 라벨에서 "데이터 수집 없음" 가능. 강력한 마케팅 포인트.

4. **결제는 App Store IAP만 사용** — 결제 서버·영수증 검증 인프라 불필요, 운영비 0원 유지. App Store Small Business Program으로 수수료 15%.
